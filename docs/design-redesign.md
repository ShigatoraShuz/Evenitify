# Eventify Redesign Direction

## Visual Direction

Eventify should feel like a premium procurement workspace for live events, not a generic SaaS dashboard.

The look should combine:

- dark, atmospheric shell surfaces
- bright ivory content planes
- sharp but not brittle card geometry
- restrained glass and glow effects
- strong typographic hierarchy
- purposeful status color, not rainbow decoration

The product should feel like a control room for event planning, vendor response, and operational oversight.

## Brand Feel

The redesign should read as:

- editorial enough to feel designed
- operational enough to feel trustworthy
- calm enough to handle dense information
- lively enough to feel event-specific

Avoid:

- default gray admin panels
- purple-heavy SaaS defaults
- stock-photo hero treatment inside authenticated surfaces
- flat card grids with no hierarchy
- tiny, low-contrast labels that force scanning

## Role Language

### Organizer

Organizer screens should feel like a planning studio and procurement command center.

Traits:

- larger context blocks
- narrative event summary
- visible progress and pipeline status
- clear next actions
- stronger emphasis on briefs, vendor selection, and booking momentum

### Vendor

Vendor screens should feel like an opportunity desk and service operations workspace.

Traits:

- request queue first
- compact but rich cards
- strong service inventory hierarchy
- clear availability and quote status
- easy access to active work, reports, and profile updates

### Admin

Admin screens should feel like an oversight cockpit.

Traits:

- dense but organized analytics
- visible queue and risk flags
- table-forward exception management
- audit trail prominence
- clear action and review paths

## Layout Principles

- Use one strong shell across all authenticated routes.
- Use asymmetrical grid layouts instead of repetitive 3-column dashboard rows.
- Give one primary action per screen and keep the rest secondary.
- Use larger section headers and tighter body copy.
- Break long surfaces into stacked blocks with different densities.
- Keep context visible while working, especially on organizer and admin screens.
- Let empty states and loading states match the final geometry of the screen.

## Component Rules

### Shell

- The authenticated shell should have a dark top frame and a layered background.
- Sidebar navigation should feel like a workspace rail, not a plain menu.
- The top bar should surface search, notifications, help, and profile without crowding the page.

### Cards

- Cards should have clearer depth, stronger border treatment, and more varied density.
- Summary cards should read like signal tiles, not generic KPI blocks.
- Feature cards should carry stronger titles, tighter metadata, and more specific actions.

### Forms

- Forms should feel structured and task-focused.
- Group related fields into clear panels.
- Make field labels stronger and more explicit.
- Use cleaner spacing between groups, not only between fields.

### Tables

- Tables should look like operational tools, not default HTML tables.
- Header rows should be clearer.
- Row hover should feel deliberate.
- Filtering and search should live close to the table, not far above it.

### Modals and Drawers

- Modals should feel like focused workspaces with stronger headers and clearer close affordances.
- Drawers should support comparison, review, and detail inspection without feeling generic.

### Empty and Loading States

- Empty states should explain the next step and offer a clear action.
- Loading states should mimic the destination layout instead of showing a detached spinner.

### Badges and Status

- Status chips should be compact, readable, and consistent.
- Use color to support meaning, not to decorate every object.

## Non-Generic Intent

The redesign should avoid the "same dashboard, different label" problem.

Each role should have:

- its own top-level hierarchy
- its own dominant information order
- its own empty-state voice
- its own dominant action pattern

But the product should still feel like one connected system.
